How Does It Work?

Finding the optimal meeting point in Prague's public transport network.

Consider you are in Prague and you want to meet with your friends at a pub. What is the optimal stop to meet? This app finds that for you!

Timetable data are scraped from the IDOS API, which uses PID (Prague Integrated Transport) schedule data.

Problem Formulation

Given a set of k starting stops S = {s₁, s₂, ..., sₖ}, we find an optimal target stop t* from all 1463 stops in Prague that minimizes travel time for all participants.

Optimization Methods

Minimize Worst Case (Min-Max)

Finds the stop where the person with the longest journey has the shortest possible travel time.

t* = argmin max(travel_time(sᵢ, t))

Minimize Total Time (Sum)

Finds the stop that minimizes the total travel time across all participants.

t* = argmin Σ travel_time(sᵢ, t)

Two-Stage Algorithm

Stage 1 — Candidate Selection

Pre-computed geographic distances and scraped travel times (~2.1M stop combinations) are used to quickly filter the top candidate stops.

Stage 2 — Real-Time Refinement

For each candidate, actual travel times are queried from the IDOS/DPP API for the specific date and time you chose. Results are sorted by the selected optimization method.

Round-Trip Support

Each participant can set a different start and end stop. The optimization considers the full round trip: travel(start → target) + travel(target → end).

Pub Discovery

For the top 5 meeting points, the app searches for nearby pubs using the Google Places API (within 500m radius) and shows ratings, review counts, and Google Maps links.
